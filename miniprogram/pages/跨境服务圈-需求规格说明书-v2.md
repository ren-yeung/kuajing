## 1. 项目概述

### 1.1 项目背景
跨境服务圈是一个连接跨境商家与服务提供商的微信小程序平台，为跨境电商从业者提供一站式服务对接能力。

### 1.2 项目目标
- 为跨境商家提供各类专业服务的查找与对接
- 为服务提供商提供展示和获客渠道
- 建立跨境服务生态圈，促进信息流通与合作

### 1.3 目标用户
- **普通用户（商家）**：需要各类跨境服务的电商从业者
- **服务提供商（商家）**：提供跨境相关服务的企业或个体（物流、支付、认证、培训等）

## 2. 双身份系统设计（核心）

### 2.1 身份体系
```
用户首次登录
    ↓
微信授权 → 注册为用户身份（user_id）
    ↓
用户提交商家资料审核
    ↓
审核通过 → 获得商家身份（merchant_id）
    ↓
用户拥有双身份：user_id + merchant_id
```

### 2.2 身份切换逻辑
- **用户版本**：默认版本，所有用户首次登录后使用
- **商家版本**：审核通过后，用户可以在"我的"页面切换至商家版本
- **版本隔离**：
  - 用户版本首页 = 服务广场（浏览服务）
  - 商家版本首页 = 需求广场（查看需求、发布服务）

### 2.3 身份权限对比

| 功能 | 用户身份 | 商家身份 |
|------|---------|---------|
| 浏览服务 | ✅ | ✅ |
| 发布需求 | ✅ | ✅ |
| 发布服务 | ❌ | ✅ |
| 查看需求广场 | ❌ | ✅ |
| 在线咨询 | ✅ | ✅ |
| 聊天功能 | ✅ | ✅ |

## 8. 技术要求

### 8.1 开发框架
- **平台**: 微信小程序
- **开发框架**: 微信原生小程序框架
- **UI组件**: 自定义组件

### 8.2 性能要求
- 首屏加载时间 < 1.5秒
- 列表滚动帧率 ≥ 50fps
- 图片懒加载
- 页面缓存策略

### 8.3 兼容性
- 支持微信小程序基础库版本 ≥ 2.0.0
- 适配不同屏幕尺寸（设计稿375px宽度）
- 适配刘海屏、底部安全区域

### 8.4 双身份技术实现
- 用户表: `user_id`, `openid`, `nickname`, `avatar`, `create_time`
- 商家表: `merchant_id`, `user_id`（关联）, `company_name`, `status`（审核状态）, `create_time`
- 身份判断: 根据用户ID查询商家表，有记录且status=approved则为商家身份
- 版本切换: 前端缓存当前身份状态，切换时刷新页面

## 9. 数据流设计

### 9.1 用户认证流程
```
微信登录 → 获取openid → 注册/登录用户表 → 返回user_id → 缓存用户信息
```

### 9.2 商家认证流程
```
用户提交商家资料 → 后台审核 → 审核通过 → 创建商家记录 → 关联user_id → 用户获得商家身份
```

### 9.3 服务发布流程
```
商家点击悬浮按钮 → 进入发布服务页面 → 填写表单 → 上传图片 → 提交 → 后台审核 → 审核通过 → 发布成功
```

### 9.4 需求发布流程
```
用户点击悬浮按钮 → 进入发布需求页面 → 填写表单 → 提交 → 需求发布成功 → 商家可在需求广场查看
```

### 9.5 即时通讯流程
```
用户点击"立即咨询" → 创建/打开会话 → 进入聊天界面 → WebSocket实时通讯
```
